# 【初心者向け】時間連動型謎解きサイト 構築手順マニュアル

このマニュアルでは、プログラミングやサーバーの知識が全くない初心者の方でも、**「アクセスした時間によって表示が変わる謎解きサイト」**を完全に構築できるよう、1から丁寧に解説します。

---

## 🛠️ システムの仕組み（なぜ「検証モード」に耐えられるのか？）

通常のWebサイトは、すべての時間帯の手がかり（HTMLデータなど）を自分のパソコンに一度ダウンロードし、JavaScriptで表示を切り替えています。そのため、プレイヤーがブラウザの**「検証（デベロッパーツール）」**を開くと、ソースコードの中にすべての時間帯の答えが丸見えになってしまいます。

今回のシステムでは、以下の工夫を行っています：
1. **フロントエンド（GitHub Pages）**：中身が空っぽの、枠組みだけのWebサイトを用意します。
2. **バックエンド（Google Apps Script / GAS）**：Googleのサーバーの中に、12パターンのデータを隠しておきます。
3. **通信**：アクセスした瞬間に、JavaScriptがGoogleのサーバーへ「今の時間のデータをください」と要求します。サーバーは時計を見て、**「現在の1時間分のデータだけ」**をWebサイトに送り返します。

これにより、プレイヤーのパソコンには**「今の時間のデータ」しか存在しない**状態になります。検証モードをいくら探しても、他の時間のデータは絶対に表示されません。

---

## 1️⃣ 【第一段階】Google Apps Script (GAS) の準備と公開

Googleが無料で提供しているサーバー（GAS）を使い、時間判定を行うプログラムを公開します。

### ステップ 1：GASを開く
1. [Google ドライブ](https://drive.google.com/) にアクセスし、ご自身のGoogleアカウントでログインします。
2. 左上にある **「＋ 新規」** ボタンをクリックします。
3. **「その他」** ＞ **「Google Apps Script」** をクリックします。
   *(※一覧にない場合は、「アプリを追加」から「Google Apps Script」を検索して追加してください)*
4. 新しいプロジェクトが開きます。左上の「無題のプロジェクト」をクリックし、`Chronos-API` など分かりやすい名前に変更します。

### ステップ 2：コードを貼り付ける
1. 最初から入力されている `function myFunction() { ... }` をすべて消して、画面を完全に空にします。
2. 同封されている `gas_code.js` の中身を**すべてコピー**し、GASのエディタに貼り付けます。
3. 画面上部にある **フロッピーディスクのアイコン（保存）** をクリックして保存します。

### ステップ 3：ウェブアプリとして公開（デプロイ）する
プログラムを世界中からアクセスできるように公開します。

1. 画面右上にある青い **「デプロイ」** ボタンをクリックし、**「新しいデプロイ」** を選択します。
2. 開いた画面の左上にある ⚙️（歯車アイコン）をクリックし、**「ウェブアプリ」** を選択します。
3. 設定項目を以下のように指定します：
   * **説明**：`初回デプロイ`（何でも構いません）
   * **次のユーザーとして実行**：`自分` (あなたのメールアドレス)
   * **アクセスできるユーザー**：**`全員`** （※ここを「全員」にしないと、サイトからデータが読み込めなくなります。非常に重要です！）
4. 右下の青い **「デプロイ」** ボタンをクリックします。

### ステップ 4：アクセスの承認（初めて行う場合のみ）
「アクセスを承認」という画面が表示された場合は、以下の手順で許可を与えてください。
1. **「アクセスを承認」** ボタンをクリックします。
2. ご自身のGoogleアカウントを選択します。
3. 「このアプリは Google で確認されていません」という警告画面が出ますが、安全ですので心配いりません。左下にある小さな文字 **「詳細」** をクリックします。
4. その下に現れる **「Chronos-API（安全ではないページ）に移動」** をクリックします。
5. 次の画面で **「許可」** ボタンをクリックします。

### ステップ 5：ウェブアプリURLのコピー
1. デプロイが完了すると、画面に **「ウェブアプリ」** という項目と、`https://script.google.com/macros/s/.../exec` で始まる長いURLが表示されます。
2. そのURLの右側にある **「コピー」** ボタンを押して、メモ帳などに貼り付けて保存しておきます。
   *(※このURLは次のステップで使います。画面は「完了」を押して閉じて大丈夫です)*

---

## 2️⃣ 【第二段階】フロントエンド（HTML/JS）の書き換え

手元にあるWebサイトのファイルと、先ほどコピーしたGASのURLを結びつけます。

### ステップ 1：GASのURLを貼り付ける
1. 同封されている `main.js` ファイルをテキストエディタ（メモ帳やVS Codeなど）で開きます。
2. 1行目にある以下の部分を探します：
   ```javascript
   const GAS_API_URL = "YOUR_GAS_WEB_APP_URL_HERE";
   ```
3. `"YOUR_GAS_WEB_APP_URL_HERE"` の部分を、先ほどコピーした**ウェブアプリのURL**に書き換えます。
   * **注意**：ダブルクォーテーション（`"`）は消さないようにしてください。
   * **例**：`const GAS_API_URL = "https://script.google.com/macros/s/XXXXX/exec";`
4. ファイルを上書き保存します。

---

## 3️⃣ 【第三段階】GitHub Pages への公開（Webサイトの一般公開）

これでコードの準備はすべて整いました。最後に、Webサイトをインターネット上に無料公開します。

### ステップ 1：GitHubアカウントの作成とログイン
1. [GitHub](https://github.com/) にアクセスし、アカウントをお持ちでない場合は作成してログインします。

### ステップ 2：新しいリポジトリ（保管庫）の作成
1. 右上の **「＋」** アイコンをクリックし、**「New repository」**（新しいリポジトリ）を選択します。
2. 設定項目を以下のように入力します：
   * **Repository name**（リポジトリ名）：`riddle-clock` など半角英数字で入力します。
   * **Public / Private**：必ず **`Public`** にしてください。（※Privateにすると無料でWebサイト化できません）
   * その他の設定は不要です。画面一番下にある **「Create repository」** ボタンをクリックします。

### ステップ 3：ファイルのアップロード
1. 画面の中央付近にある「Get started by creating a new file or **uploading an existing file**（既存のファイルをアップロード）」というリンクの、**`uploading an existing file`** の部分をクリックします。
2. パソコン上のフォルダーにある、以下の**3つのファイル**を選択してドラッグ＆ドロップします：
   * `index.html`
   * `style.css`
   * `main.js`
   *(※ `manual.md` や `gas_code.js` はアップロード不要ですが、一緒にアップロードしても動作に支障はありません)*
3. ファイルが追加されたのを確認し、画面下部にある緑色の **「Commit changes」** ボタンをクリックします。

### ステップ 4：GitHub Pages の有効化
1. リポジトリの画面上部にあるタブメニューから ⚙️ **「Settings」**（設定）をクリックします。
2. 左側のメニューから **「Pages」** をクリックします。
3. **「Build and deployment」** の中にある **「Branch」** の設定を探します。
4. デフォルトで `None` になっているプルダウンメニューをクリックし、**`main`**（または `master`）に変更します。
5. その右隣のフォルダ選択は `/ (root)` のままで、**「Save」**（保存）ボタンをクリックします。

### ステップ 5：サイトの動作確認
1. 1〜2分ほど待ってから、⚙️ **「Settings」** ＞ **「Pages」** 画面をリロード（再読み込み）します。
2. 画面上部に **"Your site is live at..."** というメッセージと共に、公開されたWebサイトのURL（例：`https://ユーザー名.github.io/riddle-clock/`）が表示されます。
3. そのURLをクリックし、無事に「時間の破片を収束中...」というローディング画面を経て、現在の時刻に応じた謎解きカードが表示されれば**大成功**です！

## 💡 トラブルシューティング（動かない場合）

### 🔴 画面が「システム未接続」のまま変わらない
* **原因**：`main.js` の `GAS_API_URL` の書き換えが正しく行われていないか、保存されていません。
* **対処**：`main.js` の1行目がご自身のURLになっているか再度確認し、GitHubに正しくアップロードし直してください。

### 🔴 「時間の接続に失敗」というエラーが表示される
* **原因 1**：GASのデプロイ設定で「アクセスできるユーザー」を「全員」ではなく「自分のみ」にしている可能性があります。
  * **対処 1**：GASのエディタを開き、右上「デプロイ」＞「デプロイの管理」を選択し、編集（鉛筆マーク）から「アクセスできるユーザー」を**「全員」**に変更して再度デプロイしてください。その際、新しく発行されたURLを `main.js` に貼り付け直す必要があります。
* **原因 2**：GASのURLが微妙に間違っている。
  * **対処 2**：コピーしたURLの末尾に余計な文字が入っていないか、`exec` で終わっているか確認してください。

### 🔴 表示される時間がずれている
* **原因**：GASプロジェクトのタイムゾーン設定が日本（JST）になっていない可能性があります。
  * **対処**：プログラム内（`gas_code.js`）で明示的に `"Asia/Tokyo"` タイムゾーンを指定して日本時間を算出しているため、基本的にはずれません。もしずれる場合は、GASの画面左側にある ⚙️（プロジェクトの設定）から、タイムゾーンが `(GMT+09:00) 日本標準時` になっているか確認してください。
